api Cliente REST (Modern JS)

Fetch API 
// Función asíncrona para obtener usuario
const getRestUser = async (id) => {
  try {
    // 1. Petición HTTP GET (URL limpia y semántica)
    const response = await fetch(`https://api.empresa.com/v1/users/${id}`, {
      method: 'GET',
      headers: {
        'Accept': 'application/json',
        'Authorization': 'Bearer token123'
      }
    });

    // 2. Validación de estado HTTP (404, 500, etc.)
    if (!response.ok) throw new Error(`HTTP Error: ${response.status}`);

    // 3. Parseo automático de JSON a Objeto JS
    const user = await response.json();
    
    // 4. Consumo directo de propiedades
    console.log(`Usuario encontrado: ${user.name} (${user.role})`);

  } catch (error) {
    console.error('Error en la petición REST:', error.message);
  }
};

// Estructura de Respuesta (JSON Ligero):
// { "id": 101, "name": "Alice Smith", "role": "Admin", "active": true }

Axios + XML 
import axios from 'axios';

const getSoapUser = async (id) => {
  try {
    // 1. Construcción manual del "Sobre" (Envelope) XML
    // Estricto cumplimiento del contrato WSDL
    const xmlEnvelope = `
      <soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/"
                        xmlns:web="http://api.empresa.com/webservice">
        <soapenv:Header>
           <web:AuthToken>token123</web:AuthToken>
        </soapenv:Header>
        <soapenv:Body>
          <web:GetUserRequest>
            <web:UserId>${id}</web:UserId>
          </web:GetUserRequest>
        </soapenv:Body>
      </soapenv:Envelope>`;

    // 2. Petición POST obligatoria con Content-Type XML
    const { data } = await axios.post('https://api.empresa.com/soap', xmlEnvelope, {
      headers: { 
        'Content-Type': 'text/xml; charset=utf-8',
        'SOAPAction': 'GetUser'
      }
    });

    // 3. Parseo Manual de XML a Objeto (Complejidad Adicional)
    const parser = new DOMParser();
    const xmlDoc = parser.parseFromString(data, "text/xml");
    const name = xmlDoc.getElementsByTagName("web:Name")[0].textContent;
    
    console.log(`Usuario SOAP: ${name}`);

  } catch (error) {
    console.error('Error SOAP:', error);
  }
};

lightbulb ¿Qué significa esto en palabras simples?

REST es como pedir una pizza por App

Es directo y ligero. Solo dices "Quiero la pizza #101" (la URL) y te llega la caja lista para comer (el JSON). No necesitas envoltorios extraños ni protocolos complicados. Es lo que usan la mayoría de las apps modernas (Instagram, Uber) porque es rápido y fácil de entender.

SOAP es como enviar una Carta Certificada

Es formal y estricto. Necesitas un sobre especial (Envelope), poner los sellos correctos (Headers) y escribir el mensaje siguiendo reglas muy rígidas. Al recibirla, tienes que abrir el sobre con cuidado para leer el contenido. Es más trabajo, pero garantiza que el mensaje llegue seguro y sin alteraciones, por eso lo usan los bancos.
