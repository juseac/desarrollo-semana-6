Métodos HTTP: Semántica y Operaciones CRUD

Entendiendo el significado semántico de los verbos HTTP y su mapeo a operaciones de base de datos.

table_chart Matriz de Características de Métodos

check_circle Soportado

Método Op. CRUD Seguro Idempotente Cuerpo

Seguro: No modifica recursos.

Idempotente: Múltiples solicitudes idénticas tienen el mismo efecto que una sola solicitud.

code Inspector: GET

Caso de Uso: Recupera una representación de un recurso. No debe contener un cuerpo de solicitud. 

GET /api/v1/students/42 HTTP/1.1
Host: api.university.edu
Authorization: Bearer eyJhbGciOiJIUz...
Accept: application/json

Éxito: El servidor encontró el recurso y lo devuelve en formato JSON. 

HTTP/1.1 200 OK
Content-Type: application/json
Cache-Control: max-age=3600

{
  "id": 42,
  "name": "Jane Doe",
  "courses": [
    "CS101", 
    "MATH200"
  ],
  "gpa": 3.8,
  "status": "active"
}

Códigos de Estado Comunes para GET

200

OK

Solicitud exitosa. El recurso se devuelve en el cuerpo.

304

No Modificado

El recurso no ha cambiado desde la última solicitud (caché).

404

No Encontrado

El recurso solicitado no existe en el servidor.

400

Solicitud Incorrecta

La sintaxis de la solicitud es inválida.
