info El protocolo HTTP es "stateless" (sin estado), lo que significa que cada petición es independiente.

Fases de Conexión

Fase Descripción
Establecimiento
Handshake TCP/IP inicial entre cliente y servidor.
Solicitud
El cliente envía headers y cuerpo del mensaje (JSON/XML).
Espera
El servidor interpreta la petición y ejecuta lógica.
Respuesta
El servidor retorna código de estado (ej. 200, 404) y payload.
Cierre
Finalización de la conexión. Liberación de recursos.
lightbulb

Nota Importante

En REST, cada petición debe contener toda la información necesaria para ser procesada. El servidor no "recuerda" el estado anterior del cliente.
