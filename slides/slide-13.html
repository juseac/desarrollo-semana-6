WSDL: Web Services Description Language

El contrato basado en XML que define el "Qué", "Cómo" y "Dónde" de un servicio SOAP.

body_system Anatomía del Documento WSDL

data_object
1. Types (Tipos - XSD)
expand_more
Define los tipos de datos complejos utilizados en los mensajes mediante XML Schema (XSD). Establece la estructura de los objetos que viajan por la red.
input
2. Messages (Mensajes - E/S)
expand_more
Descripción abstracta de los datos intercambiados. Define qué información se envía (Input) y qué se recibe (Output).
settings_ethernet
3. PortType (Interfaz)
expand_more
Define un conjunto de operaciones abstractas soportadas por uno o más endpoints. Es equivalente a una "Interface" en programación orientada a objetos.
link
4. Binding (Protocolo)
expand_more
Especifica el protocolo concreto (generalmente SOAP sobre HTTP) y el formato de los datos para las operaciones definidas en el PortType.
public
5. Service (Punto Final)
expand_more
Agrupa un conjunto de puertos relacionados y especifica la ubicación física (URL) donde se encuentra disponible el servicio web.

Generación Automática de Cliente (Proxy)

Debido a su estructura estricta, las herramientas de desarrollo pueden leer el WSDL y generar todo el código necesario para consumir el servicio.

wsimport (Java) svcutil.exe (C#) zeep (Python)
BillingService.wsdl
content_copy 
<!-- 1. TYPES: Estructura de Datos -->
<wsdl:types>
  <xs:schema targetNamespace="http://example.com/billing">
    <xs:element name="InvoiceRequest" type="xs:string"/>
    <xs:element name="InvoiceResponse" type="xs:float"/>
  </xs:schema>
</wsdl:types>

<!-- 2. MESSAGES: Entrada y Salida -->
<wsdl:message name="GetInvoiceInput">
  <wsdl:part name="body" element="tns:InvoiceRequest"/>
</wsdl:message>

<wsdl:message name="GetInvoiceOutput">
  <wsdl:part name="body" element="tns:InvoiceResponse"/>
</wsdl:message>

<!-- 3. PORTTYPE: La Interfaz Abstracta -->
<wsdl:portType name="BillingPortType">
  <wsdl:operation name="GetInvoice">
    <wsdl:input message="tns:GetInvoiceInput"/>
    <wsdl:output message="tns:GetInvoiceOutput"/>
  </wsdl:operation>
</wsdl:portType>

<!-- 4. BINDING: Protocolo (SOAP) -->
<wsdl:binding name="BillingSoapBinding" type="tns:BillingPortType">
  <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>
  <wsdl:operation name="GetInvoice">
    <soap:operation soapAction="http://example.com/GetInvoice"/>
    <!-- ... input/output definitions ... -->
  </wsdl:operation>
</wsdl:binding>

<!-- 5. SERVICE: Endpoint URL -->
<wsdl:service name="BillingService">
  <wsdl:port name="BillingPort" binding="tns:BillingSoapBinding">
    <soap:address location="http://api.example.com/v1/billing"/>
  </wsdl:port>
</wsdl:service>

Estricto

  • Estándar maduro (Enterprise)
