Cliente-Servidor

Separación de responsabilidades. La UI es independiente de los datos, permitiendo que evolucionen por separado.

Ex: React App vs Node API
do_not_disturb_on

Sin Estado (Stateless)

El servidor no guarda estado de la sesión entre peticiones. Cada request contiene toda la info necesaria.

Ex: Token JWT en Headers
bolt

Cacheable

Las respuestas deben definirse explícitamente como cacheables o no para mejorar la eficiencia.

Ex: Cache-Control & ETag
link

Interfaz Uniforme

Comunicación estandarizada. Simplifica y desacopla la arquitectura.

Ex: Verbos HTTP (GET, POST)
layers

Sistema en Capas

Arquitectura jerárquica invisible para el cliente. Mejora la seguridad y el balanceo de carga.

Ex: Proxy, Load Balancer
