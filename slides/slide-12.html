info Definición Formal

Protocolo Estándar basado en XML para el intercambio de información estructurada en la implementación de servicios web. Diseñado para permitir la comunicación entre aplicaciones independientemente del sistema operativo o lenguaje.

check_circle Ventajas

  • Transacciones ACID robustas
  • Estandarización estricta (WSDL)
  • Funciona en entornos asíncronos

warning Limitaciones

  • Verbosidad extrema (XML pesado)
  • Lento parseo comparado con JSON
  • Curva de aprendizaje alta

Estructura del Mensaje

Envelope
<Header>
Optional
<Body>
Mandatory

Ejemplo de Solicitud (CreateUser) POST /ws/users 

<?xml version="1.0"?>
<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope">
  <!-- Metadatos de autenticación -->
  <soap:Header>
    <m:AuthToken xmlns:m="http://example.org/auth">
      abc-123-xyz
    </m:AuthToken>
  </soap:Header>
  
  <soap:Body>
    <m:CreateUserRequest xmlns:m="http://example.org/users">
      <m:Username>alex_dev</m:Username>
      <m:Role>Administrator</m:Role>
    </m:CreateUserRequest>
  </soap:Body>
</soap:Envelope>

Estructura de Error

Envelope
<Header>
<Body>
<Fault>
Error Details

SOAP Fault: Mecanismo estándar para reportar errores. Incluye código, motivo y detalles específicos.

Ejemplo de Respuesta de Error HTTP 500 Internal Server Error

Error Response 
<?xml version="1.0"?>
<soap:Envelope xmlns:soap="http://www.w3.org/2003/05/soap-envelope">
  <soap:Body>
    <soap:Fault>
      <soap:Code>
        <soap:Value>soap:Sender</soap:Value>
      </soap:Code>
      <soap:Reason>
        <soap:Text xml:lang="es">El formato de los datos es incorrecto</soap:Text>
      </soap:Reason>
      <soap:Detail>
        <e:myFaultDetails xmlns:e="http://example.org/faults">
          <e:message>El campo 'Username' no puede estar vacío.</e:message>
          <e:errorcode>1002</e:errorcode>
        </e:myFaultDetails>
      </soap:Detail>
    </soap:Fault>
  </soap:Body>
</soap:Envelope>
